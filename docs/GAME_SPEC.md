**バージョン:** 0.2

## 1. 概要

### 1.1 コンセプト

プレイヤーは、グリッド上に配置された**片側の図形をヒント**にして、**反対側に左右対称な図形を並べる**パズルゲーム。

完成すると左右が鏡像のようになり、視覚的な達成感を得られる。

### 1.2 ターゲット

- 年齢層: 全年齢（子ども〜大人）
- プレイ時間: 1ステージ 30秒〜3分
- プラットフォーム: Webブラウザ（PC / タブレット / スマートフォン）

### 1.3 ゲームの流れ（1ステージ）

```
ステージ開始 → 左半分にヒント図形が表示
            → プレイヤーが右半分に図形を配置
            → 「完成」ボタンで判定
            → 正解 → 次ステージ / 不正解 → 再挑戦
```

## 2. コアゲームプレイ

### 2.1 基本ルール

| 項目 | 内容 |
|------|------|
| グリッド | **7×7** の正方形セル（奇数幅・固定） |
| 対称軸 | **縦方向の中央線**（左右対称） |
| ヒント側 | ステージ開始時に**左半分**に図形が固定配置される |
| プレイ側 | **右半分**にプレイヤーが図形を配置する |
| 対称軸上のセル | 奇数幅グリッドの中央列。図形は**そのまま**配置（反転不要） |
| 勝利条件 | 右半分の配置が、左半分の**左右鏡像**と一致する |

### 2.2 左右対称の定義

グリッド幅を `W`、中央列を `axis`（0-indexed で `floor(W/2)`）とする。

- 左側のセル `(x, y)` に図形 `A` がある場合
- 右側の対応セル `(W - 1 - x, y)` に**同じ図形 `A`** が必要
- 中央列 `(axis, y)` は左右共通のため、左と右で同じ図形であること

**例（7×7 グリッド）:**

```
左: (0, 2) = ●    →  右: (6, 2) = ●
左: (1, 3) = ▲    →  右: (5, 3) = ▲
中央: (3, 4) = ■  →  中央列（x = 3）は左右共通。同じ図形を配置
```

7×7 では中央列（x = 3）が対称軸上にあり、左右対称の概念が視覚的に理解しやすい。

### 2.3 プレイヤーの操作

| 操作 | 内容 |
|------|------|
| 図形選択 | パレットから配置したい図形をタップ/クリック |
| 配置 | 右半分の空セルをタップ/クリックして図形を置く |
| 削除 | 配置済みセルを再度タップ/クリックで図形を除去 |
| 完成 | 「完成」ボタンで正誤判定 |
| リセット | 右半分のみをクリア（左のヒントは変更不可） |
| ヒント | （オプション）正解の1マスを表示 |

### 2.4 制約

- **左半分（ヒント側）は編集不可**
- 右半分にのみ図形を配置可能
- 使用できる図形はステージごとに**パレットで指定**（不要な図形は出さない）
- 配置数に上限は設けない（空セルのままでも可）

## 3. 図形（ピース）

### 3.1 基本図形（MVP）

| ID | 名称 | 見た目 | 備考 |
|----|------|--------|------|
| `circle` | 丸 | ● | 色付き円 |
| `square` | 四角 | ■ | 色付き正方形 |
| `triangle` | 三角 | ▲ | 上向き三角形 |
| `diamond` | 菱形 | ◆ | 45°回転した正方形 |
| `star` | 星 | ★ | 5芒星 |

各図形は**色**も属性として持つ（例: 赤・青・緑・黄）。

### 3.2 図形の同一性

正誤判定では **図形の種類 + 色** の両方が一致している必要がある。

### 3.3 将来拡張（v2以降）

- 向きのある図形（矢印、L字ブロックなど）→ 鏡像で**左右反転**が必要
- 複合図形（テトリミノ風）
- カスタム図形のアップロード

> **MVP では向きのない図形のみ** とし、鏡像判定を単純化する。

## 4. ステージ設計

### 4.1 ステージ構成

```json
{
  "id": 1,
  "title": "はじめの一歩",
  "gridSize": 7,
  "palette": ["circle-red", "square-blue"],
  "difficulty": 1
}
```

| フィールド | 説明 |
|-----------|------|
| `id` | ステージ番号 |
| `title` | ステージ名 |
| `gridSize` | グリッドの一辺のセル数（正方形） |
| `palette` | 使用可能な図形リスト |
| `difficulty` | 難易度（1〜5）。図形数もこの値に連動 |

> **注:** ヒント側の図形配置は**ステージ開始時にランダム生成**される。左半分（ヒント側）のランダムな位置に、パレット内の図形を配置する。

### 4.2 難易度カーブ

| ステージ | 難易度 | グリッド | 図形数 | 特徴 |
|---------|--------|---------|--------|------|
| 1 | ★ | 7×7 | 1 | 基本操作を学ぶ |
| 2 | ★★ | 7×7 | 2 | 複数図形の対称 |
| 3 | ★★★ | 7×7 | 3 | 色と形をそろえる |
| 4 | ★★★★ | 7×7 | 5 | 中央列を含む配置 |
| 5 | ★★★★★ | 7×7 | 7 | 全種類の図形を使用 |

> **注:** ステージ数は **5** 。各ステージの難易度はステージ番号と一致（1→★、5→★★★★★）。

### 4.3 ステージ数（MVP）

- **合計:** 5ステージ（難易度1〜5を順番に体験）

### 4.4 ステージ解放

- ステージは**順番に解放**（前ステージクリアで次が開く）
- クリア済みステージはステージ選択画面から再プレイ可能

- 説明イラストはテキストの補助として、視覚的に「左がヒント・右が配置エリア」であることを示す

1. **ステージ1:** 「左の図形と同じものを右に置こう」（1図形のみ）

3. **ステージ3:** 「色も形もそろえよう」（色の違いを意識）

## 7. 技術仕様

### 7.1 技術スタック（確定）

| レイヤー | 技術 | 理由 |
|---------|------|------|
| フロントエンド | **HTML + CSS + JavaScript（Vanilla）** | シンプルな構成、ビルド不要 |
| データ | **script.js 内にステージ定義** | 単一ファイルで完結 |
| デプロイ | **静的ホスティング**（GitHub Pages 等） | サーバー不要 |

### 7.1.1 ビジュアルスタイル（確定）

- **フラット・ミニマル** デザイン
- 図形は **SVG** で描画
- 配色は **パステルカラー** を基調
- 余白を活かしたシンプルなレイアウト
- タイトル画面の説明イラストも同一スタイルで統一

### 7.2 ディレクトリ構成

```
symmetry-puzzle/
├── docs/
│   └── GAME_SPEC.md          # 本仕様書
├── index.html                # ページ構造
├── style.css                 # スタイル
├── script.js                 # ゲームロジック・ステージデータ
└── README.md
```

### 7.3 コアロジック

#### 対称判定アルゴリズム

```
function checkSymmetry(board, gridSize):
  axis = floor(gridSize / 2)
  for y in 0..gridSize-1:
    for x in 0..axis-1:          // 左半分のみ走査
      mirrorX = gridSize - 1 - x
      if board[x][y] != board[mirrorX][y]:
        return false
  return true
```

#### 不正解時のフィードバック

間違っているセルの座標リストを返し、UI で**赤くハイライト**表示する。

- 右半分（プレイ側）で配置が正解と異なるセルを特定
- 空セルなのに図形が必要な場合、または図形の種類・色が異なる場合もハイライト対象
- ハイライトは再配置または「完成」ボタン再押下まで維持
